## Introduction

To specify max height of a view. It prevents the applied value of [`height`](./height) becoming larger than the specified value of `max-height`.
The priority of `max-height` is higher than [`height`](./height) and [`min-height`](./min-height).

## Syntax

```css
/* <length> */
max-height: 120px;
max-height: 10em;

/* <percentage> */
max-height: 75%;
```

### Values

#### `<length>`

[`<length>`](api/css/data-type/length.mdx) Defines the max-height as an absolute value.

#### `<percentage>`

Defines the `max-height` as a percentage of the containing block's height.

## Difference with the Web

[mdn:max-height](https://developer.mozilla.org/en-US/docs/Web/CSS/max-height)

1. In Lynx, the priority of `max-height` is higher than `height` and `min-height`. While in Web, `max-height` overrides `height`, but `min-height` overrides `max-height`.
2. In Lynx, if `max-height` is specified smaller than [`min-height`](./min-height.mdx), [`min-height`](./min-height.mdx) will be ignored.
3. `none`, `max-content`, `min-content`, `fit-content`, `min-content` are not supported.

## Introduction

The `width` CSS property specifies the width of an element. By default, the property defines the width of the border area. If [`box-sizing`](./box-sizing.mdx) is set to `content-box`, however, it instead determines the width of the content area.

The [`min-width`](./min-width) and [`max-width`](./max-width) properties override `width`.

## Syntax

```css
/* <length> values */
width: 300px;
width: 25em;

/* <percentage> value */
width: 75%;

/* Keyword values */
width: max-content;
width: fit-content;
width: fit-content(20em);
width: auto;
```

### Values

- `auto`

  **Default value**. lynx will calculate and select a width for the specified element.

- `<length>`

  [`<length>`](api/css/data-type/length.mdx) defines the width as a distance value.

- `<percentage>`

  Defines the width as a percentage of the [containing block](https://developer.mozilla.org/en-US/docs/Web/CSS/Containing_block)'s width.

- `max-content`

  The intrinsic preferred width.

- `fit-content`

  [`fit-content`](api/css/data-type/fit-content.mdx) uses the computed size with the available space replaced by the specified argument.

- `min-content` is not supported.

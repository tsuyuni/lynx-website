## Introduction

To specify max width of a view. It prevents the applied value of [`width`](./width) becoming larger than the specified value of `max-width`.
The priority of `max-width` is higher than [`width`](./width) and [`min-width`](./min-width).

## Syntax

```css
/* <length> */
max-width: 120px;
max-width: 10em;

/* <percentage> */
max-width: 75%;
```

### Values

#### `<length>`

[`<length>`](api/css/data-type/length.mdx) defines the max-width as an absolute value.

#### `<percentage>`

Defines the max-width as a percentage of the containing block's width.

## Difference with the Web

[mdn:max-width](https://developer.mozilla.org/en-US/docs/Web/CSS/max-width)

1. In Lynx, the priority of `max-width` is higher than `width` and `min-width`. While in Web, `max-width` overrides `width`, but `min-width` overrides `max-width`.
2. In Lynx, if `max-width` is specified smaller than [`min-width`](./min-width.mdx), [`min-width`](./min-width.mdx) will be ignored.
3. `none`, `max-content`, `min-content`, `fit-content`, `min-content` are not supported.

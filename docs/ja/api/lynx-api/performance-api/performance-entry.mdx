# PerformanceEntry

`PerformanceEntry` is a base interface used to describe the complete data of performance events. All performance events inherit from and extend it, commonly including container initialization, page loading, and rendering times.

## Instance Properties

### entryType

```tsx
entryType: string;
```

Represents a string indicating the type of `PerformanceEntry`. The currently supported types are: `init`, `metric`, and `pipeline`, with the following meanings:

- **`init`**: Records performance events for the initialization process before the Lynx page is displayed.
- **`metric`**: Records performance metric events, such as FCP (First Contentful Paint), TTI (Time to Interactive), and ActualFMP (Actual First Meaningful Paint).
- **`pipeline`**: Records performance events during the rendering process.

The corresponding subclasses for each entryType value are shown in the table below.

| entryType  |                                                                                                     subclasses                                                                                                      |
| :--------: | :-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------: |
|   `init`   | [`InitContainerEntry`](./performance-entry/init-container-entry), [`InitLynxviewEntry`](./performance-entry/init-lynxview-entry), [`InitBackgroundRuntimeEntry`](./performance-entry/init-background-runtime-entry) |
|  `metric`  |              [`MetricFcpEntry`](./performance-entry/metric-fcp-entry), [`MetricTtiEntry`](./performance-entry/metric-tti-entry), [`MetricActualfmpEntry`](./performance-entry/metric-actual-fmp-entry)              |
| `pipeline` |                                                  [`PipelineEntry`](./performance-entry/pipeline-entry), [`LoadBundleEntry`](./performance-entry/load-bundle-entry)                                                  |

### name

```tsx
name: string;
```

A string representing the name of the `PerformanceEntry`. The value depends on the specific type of the `PerformanceEntry` object. The corresponding names for all subtypes are shown in the table below:

|                                    subclasses                                     |                            name                             |
| :-------------------------------------------------------------------------------: | :---------------------------------------------------------: |
|         [`InitContainerEntry`](./performance-entry/init-container-entry)          |                         `container`                         |
|          [`InitLynxviewEntry`](./performance-entry/init-lynxview-entry)           |                         `lynxview`                          |
| [`InitBackgroundRuntimeEntry`](./performance-entry/init-background-runtime-entry) |                     `backgroundRuntime`                     |
|             [`MetricFcpEntry`](./performance-entry/metric-fcp-entry)              |                            `fcp`                            |
|             [`MetricTtiEntry`](./performance-entry/metric-tti-entry)              |                            `tti`                            |
|       [`MetricActualFmpEntry`](./performance-entry/metric-actual-fmp-entry)       |                         `actualFmp`                         |
|            [`LoadBundleEntry`](./performance-entry/load-bundle-entry)             | `loadBundle`, `reloadBundle`, `reloadFromBackgroundRuntime` |
